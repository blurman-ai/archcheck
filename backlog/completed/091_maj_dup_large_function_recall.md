## Цель
Взять конкретные пары клонов, которые **нашёл NiCad, но пропустил archcheck**,
разобрать первопричину, починить безопасно (без регресса точности) и закрыть
регрессионным тестом.

## Тестовый набор (NiCad-found / archcheck-missed)
Из NiCad-эвала, корзина «only-NiCad authored»:

- **CASE A — LibreSprite `src/doc/algo.cpp`** (within-file twins):
  `algo_line` (20-101) ↔ `algo_line_float` (105-190), NiCad sim 93;
  `algo_spline_get_y` (536-612) ↔ `algo_spline_get_tan` (614-690).
- **CASE B — AetherSDR заголовки** (cross-file, декларации):
  `src/core/TgxlConnection.h` ↔ `src/core/PgxlConnection.h` (подтверждённый
  копипаст connection-класса; в комментарии прямо «Same protocol as TgxlConnection»).

## Корневые причины (замерено, не на глаз)
Воспроизведено через прямой вызов `scanForDuplication` / `extractFragments` на
реальных файлах (харнесы — `reports/raw/nicad/` нет; одноразовые в `nicad_work/`).

**CASE A — порог `maxTokens=400` режет близнецов асимметрично.**
`algo_line` = 392 токена → один цельный фрагмент `22-100`. Его близнец
`algo_line_float` > 400 токенов → фрагментатор дробит его на под-блоки
(`132-142, 147-166, 170-177`). Цельный фрагмент vs три под-блока **не
выравниваются** → клон невидим. Для *плоского* тела > cap (без вложенных `{}`,
куда спускаться) фрагментатор вообще отдаёт **0 фрагментов** — функция исчезает
из поиска. Сплайн-троица — то же самое (функции > 400 токенов, дробятся).

**CASE B — фрагментатор не эмитит фрагментов из файлов-деклараций.**
`Tgxl/PgxlConnection.h` → `frags=0`. Фрагментатор по дизайну берёт только
тела функций/управляющих блоков (`{` после `)`); декларация класса
(`class X : public QObject { signals: ... };`) тел не содержит → сравнивать нечего.

## Фикс
**CASE A — `maxTokens` 400 → 600** в `FragmentOptions`
([include/archcheck/scan/duplication/fragmenter.h](../../include/archcheck/scan/duplication/fragmenter.h)).
600 держит цельными функции до ~120 строк. Это и есть лечение асимметрии:
оба близнеца остаются цельными и матчатся.

**CASE B — НЕ чиним (осознанно), и чинить нечего.** Оказалось: реальный
копипаст `Tgxl/Pgxl` archcheck **и так ловит — в реализации**
(`PgxlConnection.cpp:111-130 ↔ TgxlConnection.cpp:121-150`, w=0.78 в baseline;
после cap=600 — `74-131 ↔ 79-151`, w=0.88). NiCad дополнительно показал **тот же**
дубль во второй раз — в *декларации* (`.h`). То есть actionable-находка не
терялась; «промах» был только в избыточном заголовочном представлении.
Фрагментировать тела деклараций вернуло бы ровно тот «expected-pattern» шум, что
в эвале — FP: 9 из 11 «only-NiCad authored» в AetherSDR это Qt-заголовки виджетов,
похожие по дизайну. Ловить дубль-декларацию ценой 9 FP — против философии
archcheck (точность под гейт). Декларационный копипаст — вне скоупа; не баг.

## Как работает
Единственный рычаг — `FragmentOptions::maxTokens`. Фрагментатор спускается по
вложенным `{}` только когда блок шире cap; при cap=600 функции до ~120 строк
остаются цельными единицами сравнения, поэтому пара near-identical близнецов
одного размера выравнивается в один клон вместо «цельный vs набор под-блоков».
Сопутствующий vendored-фикс расширяет классификацию путей: курируемый
`kVendoredLibDirs` ловит multi-file либы, лежащие под своим каталогом без
обёртки `third_party/` — это держит корпусную точность стабильной при
расширении окна фрагмента.

## Ключевые решения
- **cap 400 → 600, а не «фрагмент = вся функция»**: прагматичный шаг, лечит
  замеренную асимметрию здесь и сейчас; полное решение — за #062.
- **CASE B (декларации) не чиним**: actionable-копипаст и так ловится в
  реализации; фрагментация тел деклараций вернула бы 9 FP (Qt-заголовки) —
  против гейта точности archcheck. Декларационный дубль — вне скоупа, не баг.
- **vendored-фикс вынесен в отдельный коммит**: иная причина (классификация
  путей), хоть и сделан в одной сессии ради стабильности корпусных дельт.
