# [SCAN] исключать vendored/generated код из анализа (включая «скрытых» вендоров)

**Дата создания:** 2026-06-14
**Статус:** wip
**Дата старта:** 2026-06-19
**Модуль:** SCAN
**Приоритет:** major (по применимости — близко к release-блокеру)
**Сложность:** unknown
**Блокирует:** demo/контрибьюшн на чужом коде (#1/#3); применимость на проектах с bundled-deps
**Заблокирован:** —
**Related:** #088 (corpus FP), #125 (extensionless — другой слой слепоты), #129 (единый проход/общий view — этот предикат вставляется туда), goodfirstissue dogfood (`experiments/goodfirstissue_dogfood/`)
**Верификация:** #131 (реклон + перечисление: supercollider 2929→69; bpftrace 51→4 — SF.8 4→3, 3 остаточных vendor-FP НЕ докрыты; newsboat 6→6 без изменений; без over-exclude своего кода)

## Цель

archcheck должен **не анализировать вкомпиленный чужой код** (vendored deps) и
**генерённые файлы** — иначе на любом реальном C++-проекте с bundled-зависимостями
результат тонет в чужом шуме. Текущий фильтр (`kVendoredExact` по basename +
немного путей в `file_classification.h`) не покрывает реальные случаи.

## Доказательства (goodfirstissue dogfood, 2026-06-14)

Прогон archcheck по C++-репам с goodfirstissue вскрыл проблему на 3 из 8 реп,
причём масштаб разный:

| Репа | Что | Эффект |
|---|---|---|
| **supercollider** | `external_libraries/boost/` (4320 .hpp вкомпиленного Boost) + ICU | **~2900 нарушений, >95% — Boost/ICU FP**; реальные TP (QtCollider) утонули |
| **bpftrace** | `src/stdlib/include/` (заголовки ядра UAPI, SPDX GPL-2.0) + `bpf_assembler.h` («copied from bcc») | SF.8 ×4 FP |
| **newsboat** | `filter/Parser.h` + `Scanner.h` (генерённые Coco/R) | SF.8 ×2 FP |

Вывод supercollider — приговор применимости: **без фильтра archcheck на проекте с
bundled-deps непригоден** (5-10 реальных находок vs тысячи vendor-шума).

## Почему наивный путь-фильтр НЕ решает (замер по корпусу)

Прогон по корпусу `~/oss` (1686 реп) на «скрытых» вендоров — папки с
вложенным LICENSE/COPYING ниже корня репы:

- **2183** вложенных LICENSE всего → **945** ловятся токенами
  (`third_party|vendor|external|3rdparty|deps|contrib|...`), но **1238 (57%) — НЕТ**.
- После отсева шума (license-коллекции, docs, debian) — **515 папок-с-кодом-и-LICENSE
  вне токенов в 168 репах**: реальные вкомпиленные либы в непредсказуемых путях.
- Опознанные по имени (наивный токен-фильтр их пропустит): **imgui, eigen-3.3.7,
  Eigen, CMSIS, hidapi, zlib, STM32H7xx_HAL_Driver, miniz, ggml, fmt, tbb, glfw,
  googletest-src**.

То есть имена vendor-папок непредсказуемы (`external_libraries`, `src/stdlib/include`,
`libs`, имя самой либы) — **нужны сигналы, не зависящие от имени папки.**

## Как фильтровать — слои (дёшево → дорого, ранний выход)

**1. `.gitmodules` — почти бесплатно и точно.**
Сабмодуль = чужой репозиторий по определению. Парсить `.gitmodules`, исключать
перечисленные пути. Большой класс даром.

**2. Вложенный `LICENSE`/`COPYING` в глубине дерева + рядом есть код.**
У своего проекта лицензия в корне. Свой LICENSE в `external_libraries/boost/`,
`src/stdlib/include/.../COPYING` → вкомпиленный отдельный проект. Дёшево (stat).
ВАЖНО (из замера): квалификатор «+ в каталоге есть .h/.cpp» обязателен — иначе
ловит license-коллекции/docs/debian (шум: 1238 → 515 после фильтра по коду).

**3. Токены пути** — `third_party|3rdparty|vendor|vendored|external|extern|deps|
contrib|submodule|subproject|/ext/|/import/` — как один из слоёв (ловит ~43%
вложенных лицензий), не единственный.

**4. Несовпадение копирайта/SPDX с модой репы** (читает шапки).
Мода copyright по репе; файлы с ЧУЖИМ копирайтом/SPDX → vendored. Генерализует
«copied from» без явной папки (bpftrace UAPI = GPL-2.0 Linux при BSD-репе).
+ маркеры шапки: `copied from`, `imported from`, `DO NOT EDIT`, `@generated`,
`autogenerated`, `// Generated by` (заодно класс generated).

**5. Граф-остров (структурный, граф уже строится).**
Vendored либа = плотный подграф, в который проект ВКЛЮЧАЕТ, но который обратно в
проект НЕ включает (асимметрия направления). Для каталога-кандидата:
incoming(project→subtree) > 0, outgoing(subtree→project) == 0, подграф крупный/
связный → vendor. `external_libraries/boost/` в supercollider = чистый сток.
Не читает содержимое, но нужен порог; тяжелее остальных.

**Рекомендация:** слоями с ранним выходом: `.gitmodules` → токены → вложенный
LICENSE+код → (для оставшихся) копирайт-мода / граф-остров.

## Генерённые файлы (смежный под-класс)

Отдельно от vendor, но решается теми же маркерами шапки (`@generated`, `DO NOT EDIT`,
`Generated by`) + известные пары/паттерны: Coco/R `Parser.h`/`Scanner.h` (newsboat),
`*.pb.h/.pb.cc` (protobuf), `moc_*`/`ui_*`/`qrc_*` (Qt), `*.tab.{h,c}`/`*.yy.{h,c}`
(bison/flex), `*_generated.h` (flatbuffers).

## Немигрированные правила всё ещё не исключают generated (2026-06-19)

Ревью #129-архитектуры нашло: 4 правила на «path-trio» (god_file_growth,
defect_attractor, satd, test_co_evolution) классифицируют content-free git-пути и
НЕ прошли через `AuthoredScope`/`SourceSnapshot`. У `god_file_growth` оба call-site
(`calculateP75`, `detect`) несли комментарий «exclude vendor, test, **and
generated**», а код делал лишь `pathHasVendoredDir || pathHasTestDir ||
isTestBasename` — ни generated, ни `isVendoredBasename`. Комментарии приведены к
правде (коммит docs/comment), но **сам gap остаётся**: `isGeneratedPath` —
content-free (пути `_wrap.cpp`/`.pb.h`/`moc_`…), он же подходит этим правилам без
чтения контента. Когда ляжет path-слой #127 — добавить `isGeneratedPath` (и,
вероятно, `isVendoredBasename`) в общий path-предикат этих 4 правил. Это
поведенческое изменение default-правил → нужны фикстуры + отдельный коммит, не
тихая правка.

## ⚠️ Прозрачность обязательна (принцип «no silent caps»)

Печатать `excluded N files as vendored/generated (reason: gitmodules / nested-LICENSE
/ copyright-mismatch / island)`. Тихое исключение однажды съест реальный код проекта
и мы не заметим — ровно тот скептик-риск, что уже всплывал. Возможен флаг
`--no-vendor-exclude` для отладки.

## Проверка (фикстуры обязательны)

- [ ] `fixtures/vendor_exclusion/pass_gitmodules/` — путь из `.gitmodules` исключён
- [ ] `fixtures/vendor_exclusion/pass_nested_license/` — `libs/foo/LICENSE`+код исключён, корневой LICENSE — нет
- [ ] `fixtures/vendor_exclusion/fail_own_code/` — свой код с нарушением НЕ исключён (анти-over-exclude)
- [ ] `fixtures/vendor_exclusion/pass_generated_marker/` — файл с `// @generated` исключён
- [~] regression на реальных (факт #131): supercollider 2929 → **69** ✅; bpftrace SF.8 4 → **3** (не 0 — 3 остаточных vendor-FP: `bpf_assembler.h` + 2 UAPI с голым GPL-2.0); newsboat 6 → **6** (не изменился — Coco/R этой версии без текстового баннера). Цель «десятки» по supercollider достигнута; хвосты bpftrace/newsboat — отдельному калиброванному слою.

## Самопроверка

Главный риск — OVER-exclude (съесть свой код). Проверять перечислением: что именно
исключили и почему; считать own-vs-vendor на 2-3 реальных репах вручную. «Стало 0
нарушений» после фильтра — повод проверить, не выкинули ли лишнего.

## Попытка copyright-owner mismatch и почему откатили (2026-06-19)

Пробовал самый дешёвый слой копирайт-мисматча поверх #129: вынести из заголовка
«первый токен правообладателя» (`copyrightOwnerToken`), взять доминантного владельца
репы (>50%), исключать файлы с другим владельцем. На rmm это сработало (rapidcsv
Berggren ≠ NVIDIA-доминант → clone+cplx 1/3→0/0). НО **200-коммитная проверка поймала
over-exclusion** — откатил:

- **mne-cpp** (BSD-self-licensed): копирайт по **именам авторов** («Copyright (C) 2014,
  Christoph Dinh, Lorenz Esch» — разные на файл) → нет доминантного владельца → fallback
  исключил собственные full-BSD-баннерные файлы → cplx 4→0, edges 3→0. Зануление
  собственного кода.
- **micropython** `ports/alif` (OpenMV LLC — контрибьютор-порт в дереве) +
  гетерогенный copyright в `ports/` → edges 4→0.

**Вывод:** «другой правообладатель» ≠ «вендор». Контрибьюторы, порты, со-авторы, поглощённый
код имеют другой copyright, но это **собственный** код проекта. Наивный owner-mismatch
тихо теряет recall — для архитектурного чекера это худший класс ошибки. **Слой требует
калибровки на корпусе** (тот самый список выше: `.gitmodules` → nested-LICENSE+код →
copyright-mismatch ТОЛЬКО вместе с другими сигналами, не в одиночку → граф-остров). Не
шипить копирайт-мисматч как единственный сигнал. #129 (dominant-banner-bool) — безопасный
baseline, rmm под ним остаётся (нужен этот калиброванный слой).

## Evidence-based переприоритезация раунда (2026-06-19, старт wip)

Стартовали со слоя `.gitmodules` (рекомендация в «слоях» выше) → **отвергли по проверке кода**:
dogfood-клон нерекурсивный (`scan_repo.sh:17` — `git clone --depth 1 --filter=blob:none`, без
`--recursive`), поэтому boost/ICU/UAPI закоммичены в дерево **напрямую**, а не gitlink'и; вдобавок
diff/memory-режим структурно слеп к сабмодулям (`git ls-tree -r` не спускается в gitlink, файлов
сабмодуля нет в дереве родителя → `list()` их не видит, исключать нечего). Корпус-замер: `.gitmodules`
есть у 178/1689 реп, но **населённых C++ путей сабмодулей — 3** (обычный клон сабмодули не качает).
`.gitmodules` не дотягивается ни до одного из трёх dogfood-кейсов → демотирован в отдельный мелкий
задаток (поможет CI с рекурсивными сабмодулями, не более).

**Что реально бьёт по кейсам (проверено по файлам):**
- **supercollider** — контейнер-токен `external_libraries` (нормализ. `externallibraries`, в
  `kVendoredDirNames` его НЕТ; есть только `external`/`externals`). Ловит и `external_libraries/boost/**`,
  и `external_libraries/icu/unicode/*` разом, name-independent, в disk- и diff-режиме. Остаток
  `SCDoc/SCDoc.tab.hpp` — bison C++, ловится расширением `isGeneratedPath` (`.tab.hpp/.cpp/.cc`;
  сейчас только `.tab.c`).
- **newsboat** `filter/Parser.h`/`Scanner.h` (Coco/R) — только по **content-баннеру** Coco/R/generated.
  По basename матчить НЕЛЬЗЯ (`Parser.h`/`Scanner.h` — сплошь свой код). При реклоне подтвердить баннер;
  нет баннера → остаток (не форсировать).
- **bpftrace** — `posix_types.h` и ~96/118 файлов под `src/stdlib/include` ловятся маркером
  `Linux-syscall-note`. **Остаток — 3 файла: `bpf_assembler.h` (держался на `copied from`; маркер
  выкинут #131 → вернулся как FP), `compiler-clang.h`/`errno.h` — голый однострочный `SPDX GPL-2.0`,
  без отличимого маркера и БЕЗ вложенного LICENSE.**
  Проверено: в `src/stdlib/` вложенного LICENSE нет (только корневой + `tests/.../LICENSE`), а
  однострочный SPDX отвергнут как сигнал решением #081. Значит **nested-LICENSE для bpftrace бесполезен.**

**Решение по скоупу раунда: A + B.**
- **A** — токен `external_libraries`.
- **B** — content-маркеры generated/copied через новый `hasGeneratedHeader` в `AuthoredScope` +
  path `.tab.h` в `isGeneratedPath`. Набор маркеров финализирован замером (см. ниже): только
  **distinctive** `@generated` / `coco/r` / `linux-syscall-note` (3). Прозо-рискованные
  `generated by` / `imported from` / `do not edit` исключены сразу; `copied from` выкинут отдельно
  (#131, см. ниже) — все ловили свой код.

**Отложено (НЕ в этот раунд):** generic nested-LICENSE+код (broad-corpus value, но не служит
dogfood-цели и несёт over-exclude риск — отдельный корпус-калиброванный раунд + enumeration не-project
файлов, т.к. LICENSE/COPYING не попадают в `list()` из-за `hasProjectExtension`); bpftrace-остаток
`compiler-clang.h`/`errno.h` (нужен `stdlib/include` path-токен или SPDX-GPL-mismatch — рискованно);
`.gitmodules`-слой; transparency-строка (единым механизмом по всем причинам, отдельным коммитом).

**Пойман и устранён over-exclude баг (тот самый главный риск):** первичный набор маркеров включал
прозо-рискованные `generated by` / `imported from` — они съели СВОИ bpftrace-хедеры `bpfbytecode.h`
(«bytecode generated by bpftrace») и `functions.h` («imported from pre-compiled BPF programs»). Поймано
скептик-перечислением (не агрегатом). Добавлены anti-over-exclude регрессии в unit-тест на эти точные
прозо-строки.

**`copied from` тоже выкинут (#131-ревью):** корпус-проверка — `copied from` совпадает 82/91 вне
любого vendor-сигнала, с подтверждённым own-FP на **libjxl**; собственное дерево archcheck спасало
лишь случайность offset >2000 байт. Уникального покрытия — 1 файл (`bpf_assembler.h`), не стоит
прозо-поверхности. Итоговый набор — 3 distinctive-маркера. Цена: bpftrace `bpf_assembler.h` вернулся
в SF.8 (остаток, см. выше).

**Итог раунда:** supercollider (приговор применимости) закрыт — 2929→69, весь шум external_libraries
снят без потери своего кода. bpftrace — сильно лучше (51→4), но 3 остаточных vendor-FP (UAPI/copied)
не докрыты безопасными сигналами. newsboat — остаток (нет безопасного name-independent сигнала).
Главный выигрыш по применимости достигнут; «хвосты» bpftrace/newsboat — для отдельного калиброванного
слоя (`stdlib/include` path-токен / SPDX-mismatch / COCO-guard), не форсировать.
