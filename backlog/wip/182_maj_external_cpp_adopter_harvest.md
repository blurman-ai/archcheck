## Goal

Build a corpus of **externally-validated C++ agent-adopters** from the three published repo
lists (and widened detection markers) named in #180 §"External literature & new assets",
deduped against our own funnel. Three payoffs, in priority order:

1. **Defensibility** — externally-curated adopters blunt the "your agent-detector is homemade /
   you cherry-picked" review objection against #180.
2. **Replication** — run archcheck on the EXACT C++ repos Agarwal/Mao drew conclusions from and
   compare to their published claims (strongest referee defense; #180 §"must engage").
3. **Coverage** — recover repos our ≤100-commit screening misclassified as non-agentic, and add
   net-new adopters. Bot-account-only detection undercounts Claude Code ~30× (census
   arXiv:2606.24429) — widened markers close that.

**Overlap policy (author, 2026-07-11):** a candidate already in our corpus is **tagged**
externally-validated and re-homed into this corpus, **never dropped and never counted twice**.

Replit/Cursor/aider/Codex users. Add, from the census + Robbes:

marker-vs-first-use lag ≤2 wk) before its adopters enter #180's panel. Markers that fail stay
in a diagnostic column.

## Phase B3 result (widened-marker LOCAL sweep — done 2026-07-11)

Swept all 2,192 local clones (`~/oss` + `~/oss_agentic` + `~/oss_adoption`) with the widened set
(`sweep_widened_local.py` → `widened_local_ledger.tsv`; pure local git, no network). Robbes
"Perils" (arXiv:2601.18345, 8 pitfalls) read first; its lessons changed the marker set mid-flight.

- **300 repos carry a strong widened marker — but 249/300 are `.github/copilot-instructions.md`**,
  the ONE real gap in our original 4 (we had the symmetric CLAUDE.md but not copilot-instructions).
  Cursor (`.cursorrules`/`.cursor/rules` 33), aider (`(aider)`+`.aider.conf.yml` 4), `.claudeignore`
  2, `.windsurf` 1 — the long-tail agents barely appear in serious C++.
- **Audit (the Robbes gate) killed the message-signature marker:** `codex-cli`/`Generated by Codex`
  matched topic MENTIONS, not agent commits ("ai: subscription CLI providers — claude-cli and
  codex-cli", "Fix MPS autocast", "Remove use of TBB") → demoted to a diagnostic column; only
  config-file + `(aider)`-author markers trusted. copilot-instructions hits spot-checked genuine
  (z3's was authored by the Copilot agent itself, PR #7766).
- **Yield, honestly decomposed** (`widened_eligible_b3.tsv`): of 295 trusted-marker repos, 142 pass
  the #180 eligibility gate — but 101 were already `agentic=1` (not a B3 discovery). B3's UNIQUE
  contribution:
  - **8 solid recoveries** — in-funnel repos we'd marked `agentic=0`, all substantial long-lived
    C++: lammps, htcondor, librealsense, FarManager, ezEngine, areg-sdk, grape, RisohEditor
    (12 recovered, 4 correctly dropped by the ≥12-wk gate — e.g. ppsspp's marker is 1.4 wk before HEAD).
    **Of the 8, 4 are already in the ext182/#180 pools (ezEngine, lammps, htcondor, librealsense
    — cross-validation: two independent routes agree they adopted), and only 4 are net-new to
    every route: FarManager, grape, areg-sdk, RisohEditor.**
  - **33 out-of-funnel C++ candidates** (locally cloned, never screened), 27 net-new — LOWER
    confidence: unscreened, and ≥1 is a known born-agentic trap (makr-code/ThemisDB, the #180
    domination lesson). Need the CORPUS_CRITERIA gate before any use, not blind inclusion.
  - **B3 net-new contribution = 4 screened recoveries + 27 unscreened C++ candidates = 31**, of
    which only the 4 recoveries are high-confidence; the 27 await CORPUS_CRITERIA.
- **Verdict: widened markers add little in serious C++ — `copilot-instructions.md` is the only
  material gap (+8 solid recoveries).** Confirms B1 (the "missed" class was a ≤100-commit sampling
  artifact, not a marker-set gap). The Cursor/aider/Replit/Windsurf long tail is a JS/web
  phenomenon, thin in C++ — itself a defensible marker-completeness result for #180's write-up.
