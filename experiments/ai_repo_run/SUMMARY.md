# Куда уехала архитектура за год агентской разработки

> Рабочее название статьи. Краткий отчёт — одна страница. Полная версия —
> `DRIFT_RUN_REPORT.md`.

## Что сделали

- **Инструмент (archcheck):** по дереву исходников C++ строит граф включений и
  считает архитектурные метрики — **дублирование кода** и **циклы/связность
  графа**. Работает **zero-config** (на дефолтах, без файла настроек).
- **Эксперимент:** прогнали обе метрики по **истории** 85 реальных C++-проектов
  из датасета AIDev (где использовались ИИ-агенты), по 25 точек на проект —
  смотрели, как метрики меняются от первого коммита к последнему.

## Что выяснили (реальные числа)

| Метрика | Результат по 85 проектам |
|---|---|
| Дублирование РАСТЁТ | **39 проектов** (медиана +8 п.п., макс +69) |
| Дублирование стабильно | 28 |
| Дублирование СНИЗИЛОСЬ | 18 |
| Появились циклы в графе (было 0) | **15 проектов** |
| Циклов стало больше | 20 |

- **Дрейф реален и измерим** — но виден только **по истории**, не по срезу.
- **Дрейф локализуется до коммита.** Пример: в acts (CERN) «косметический»
  рефактор `233ba90c` добавил по строке `#include` в 70 файлов — и создал **40
  циклических зависимостей за раз**. На ревью диф выглядит безобидно.
- **Дрейф обратим.** 18 проектов снизили дублирование рефакторингом; типовой
  приём — вынос общего кода в общий заголовок/базовый класс (словарь Lakos).
- **Копипаст и граф деградируют независимо** — нужны обе метрики.

## ⚠️ Что НЕ доказано (честная рамка)

**Гипотеза «ИИ ускоряет дрейф» — НЕ проверена этим прогоном.**

- Мы мерили дрейф **в проекте целиком**, не отделяя ИИ-код от человеческого.
- Попытка атрибуции по git дала: из 11 коммитов-виновников **0 с ИИ-маркерами**.
  Но это **не «ИИ не виноват»** — это «**git не годится для атрибуции**»: метка
  `Co-Authored-By` ставится непоследовательно (FastLED: 116 коммитов с
  `Co-Authored-By: Claude`, но виновники дрейфа — без неё).
- Итог: **доказано, что инструмент ловит дрейф. НЕ доказано, что причина — ИИ.**
  Это разные вопросы, их нельзя смешивать.

## Главный вывод

Архитектура деградирует **в конкретных, находимых коммитах** — и archcheck их
ловит независимо от того, кто автор. Это и есть позиционирование инструмента:
**не «кто написал», а «какой коммит сломал».**
