# vendor_exclusion fixtures (#153 item 1, split from #127)

On-disk trees for the vendor/generated exclusion layer (`AuthoredScope` +
`file_classification.h`: path/basename, license-banner, `@generated` banner,
minified-content). Consumed by
`tests/integration/scan/vendor_exclusion_fixtures_test.cpp`.

- `pass_nested_license/` — `libs/foo/` is a vendored subtree under a
  **non-vendor-named** directory (neither `libs` nor `foo` is a curated vendor
  dir token): `render.cpp` carries a full MIT license banner in its header, so
  it is excluded on the content-banner layer alone. The companion `LICENSE`
  file is scanned too (it is a regular file under the tree) and also carries
  the same banner text, so it counts as a second banner-carrying candidate.
  `own/` (3 files, no banner) keeps the ratio at 2 banners / 5 candidates —
  well under the 50% dominant-license guard — so the exclusion does not
  spread to the rest of the tree. Verified directly against `AuthoredScope`
  (not just asserted): `own/*` → `excluded=false`, `libs/foo/LICENSE` and
  `libs/foo/render.cpp` → `excluded=true`.
- `fail_own_code/` — anti-over-exclude regression. `a.h` is genuine own code
  with a real SF.7 violation (`using namespace std;` at global scope) and
  carries the exact prose that historically ate real code when a looser
  marker was tried (`bytecode generated by bpftrace`, `imported from
  pre-compiled BPF programs` — see file_classification.h's `hasGeneratedHeader`
  comment). It must stay in scope and the violation must still be reported.
- `pass_generated_marker/` — `gen.cpp` opens with a standalone `// @generated`
  banner tag; excluded regardless of the license-banner ratio (per-file truth).
- `pass_minified_data/` — `blob.cpp` is one 12+ KB line (a byte-array data
  blob); excluded as minified/embedded content (#147), not vendor or license
  related.
