# Boolean accretion: когда «+N булей» это ок, а когда смелл

**Тезис заметки.** Метрика `n_bool_field_added` (#135: нетто-прирост depth-0 bool-полей
в *пред-существующей* структуре за коммит) меряет **накопление булевого состояния —
нейтральный феномен, а не приговор «плохой код»**. По одному добавленному булю на глаз
нельзя сказать «вот он, дрейф». Добавленные були делятся на два визуально неотличимых класса:

- **Легитимные, ортогональные** — независимые бинарные свойства; любая их комбинация
  валидна. Честное доменное моделирование, не дефект.
- **Boolean-blindness смелл** — несколько булей кодируют **взаимоисключающие или
  коррелированные** состояния, и невалидная комбинация *представима в типе*, хотя
  недостижима по смыслу. Просится `enum` / sum type. Вот это и есть constraint decay
  ([constraint_decay.md](constraint_decay.md), Dente et al., EURECOM 2026).

## Эталонный пример — оба класса в одном коммите

**GregorGullwi/FlashCpp** (hobby C++ компилятор), commit `b968556`,
`src/AstNodeTypes.h`, класс `LambdaExpressionNode`. Один коммит долил 3 bool-поля
(было 1 → стало 4 — настоящий дрейф в живую структуру):

```cpp
bool is_mutable_;     // ┐ независимы, ортогональны — лямбда может быть
bool is_noexcept_;    // ┘ любой комбинацией mutable × noexcept        → ВАЛИДНО
bool is_constexpr_;   // ┐ ВЗАИМОИСКЛЮЧАЮЩИЕ: в C++ лямбда не может быть
bool is_consteval_;   // ┘ одновременно constexpr И consteval          → СМЕЛЛ
```

- `is_mutable_` / `is_noexcept_` — два независимых свойства, все 4 комбинации валидны.
  Это **не** дефект, это корректная модель синтаксиса лямбды.
- `is_constexpr_` + `is_consteval_` кодируют трёхзначное состояние (ни то / constexpr /
  consteval) двумя булями. Комбинация `(true, true)` представима, но в C++ недостижима:
  лямбда не бывает разом `constexpr` и `consteval`. Инвариант «не оба сразу» нигде не
  выражен в типе — держится на дисциплине. Каноничный фикс — `enum class
  LambdaConstevalKind { None, Constexpr, Consteval }`.

То есть **в одном и том же коммите рядом и валидные були, и смелл** — метрика accretion их
не различает и не должна.

## Следствия

1. **Для продукта (#090, отложено).** bool-дрейф подавать как **advisory**, не error:
   сам факт `+N булей` не доказывает дефект. Иначе правило завалит ложными
   срабатываниями на честном доменном моделировании.
2. **Для интерпретации (#119).** Судья «накопление булей = прокси реальной беды?» —
   **корреляция** (растёт ли accretion вместе с copy-paste / сложностью), а не raw-счёт.
   Метрика — нейтральный квант, вывод даёт корреляционный разбор.
3. **Для витрины / findings.** Не клеить ярлык «плохой дрейф» на каждый коммит с
   `+N булей`. Класс «boolean-blindness» (взаимоисключающие состояния N булями) —
   отдельный, более сильный сигнал, чем просто accretion; отличить его умеет только
   доменное знание (как `constexpr ⊥ consteval`) или корреляция.
